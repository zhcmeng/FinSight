# 网络工具

## 描述
本技能只覆盖 Trae 自带搜索/网页读取无法稳定处理的场景：PDF 正文提取、强动态页面渲染、反爬导致的内容缺失。

## 何时使用
- Trae 自带搜索已能搜到链接，但网页读取拿不到正文或被反爬拦截
- 目标内容是 PDF，或网页为强动态渲染导致静态抓取不完整

## 指令

优先使用 Trae 自带能力完成：

- 先用内置搜索获取链接
- 再用内置网页读取提取正文

只有在以上方式拿不到可用正文时，才允许使用本技能提供的终端工作流。

**严禁**自行编写 Python 脚本。你 **必须** 通过终端（Terminal）调用提供的 CLI 适配器。

### 1. 网页抓取 (Crawler)
- **抓取网页或 PDF 正文（内置网页读取失败时使用）**:
  ```bash
  python src/trae_tool_adapter.py web --type click --urls "<URL1>,<URL2>" --task "<任务描述>"
  ```

### 2. 动态搜索补位 (Search)
- **强动态/反爬场景下补充搜索结果（仅在内置搜索结果明显不足时使用）**:
  ```bash
  python src/trae_tool_adapter.py web --type search_bing --query "<搜索词>"
  ```

## 最佳实践
1. **先内置后补位**: 先用内置搜索与网页读取，确认确实缺正文再用本技能。
2. **任务导向**: 调用 `click` 时提供清晰的 `task` 描述，直接说明要抽取的字段与范围。
3. **原文优先**: 能抓到公告/政策原文就不要依赖二手解读页面。
